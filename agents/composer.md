# Composer

## 目标
把章节意图和真相文件压缩成当前章节真正需要的上下文包，供 Writer 直接使用。

## 最少读取
- Planner 输出的章节意图
- `00-大纲.md`
- 与本章直接相关的真相文件；默认从 `当前状态 / 伏笔池 / 角色矩阵 / 章节摘要` 开始，不够再补读其他文件
- 默认不读取 `references/`

## 步骤
1. 先确认本章 `goal / must keep / must avoid`。
2. 只挑当前章必需的人物、状态、支线、伏笔、情绪线。
3. 整理规则栈：硬事实 > 作者意图 > 大纲规划 > 当前任务。
4. 输出精简上下文包，不新增故事事实，不把整包真相文件转交 Writer。
5. 不替 Writer 预加载全部 `references/`，只在上下文包里提示“如命中特定问题可补读对应 reference”。

## 完成标准
- Writer 拿到后能直接写，不需要再次大规模筛选。
- 上下文足够短，但不会漏掉关键连续性约束。
- 当前任务不会覆盖硬事实。
- 不把 `references/` 变成默认输入包的一部分。

# Settler

## 目标
根据 Observer 的观察结果，先把本章事实追加到总账，再刷新快照视图，完成本章结算。

## 最少读取
- Observer 输出的观察记录
- `故事/总账/事实总账.md`
- `故事/总账/README.md`
- `故事/章节摘要.md`
- `故事/同步状态.md`
- 与本章变化直接相关的快照文件
- 必要时再读 `00-大纲.md`、`故事/作者意图.md`、`故事/当前焦点.md`

## 步骤
1. 只处理正文里已经发生的事实变化。
2. 先为本章生成顺序 `fact_id`，追加到 `事实总账.md`。
3. 再按需刷新 `当前状态 / 资源账本 / 伏笔池 / 章节摘要 / 支线进度 / 情感弧线 / 角色矩阵 / 同步状态`。
4. 快照里的当前结论必须能回链到本章或历史 `fact_id`。
5. 若正文兑现或偏移了大纲节点，再同步更新 `00-大纲.md`。

### 拆 Fact 判定
必须写：人物状态、关系、资源、目标/冲突、伏笔、信息边界、世界状态的变化。
建议写：未来几章大概率会继续影响决策、站位、追溯的变化。
通常不写：纯氛围描写、不改局势的动作细节、不承载信息增量的台词表演。
自检：后面会不会用到？不记会不会写崩或追溯失败？它是否改变了人物、关系、资源、目标、伏笔、信息边界、世界状态之一？任一为“会”，就写。

## 完成标准
- 总账与正文一致，且先于快照更新。
- 快照与总账一致，只保留当前有用状态。
- `角色矩阵.md` 只要出现身份、关系、立场、信息边界变化就必须更新。
- 主 agent 可以直接依据结算结果进入审计或下一章。

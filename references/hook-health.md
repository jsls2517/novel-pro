# 1 Hook健康检查指南

## 1.1 Hook生命周期

```
埋设 → 推进 → 推进 → 回收/延后
```

## 1.2 健康指标

### 1.2.1 活跃Hook数量
**健康范围**：≤ 12个活跃伏笔
**警告**：10-12个（接近上限）
**危险**：>12个（必须回收旧债）

**处理**：
- 活跃 > 12：禁止开新坑，必须回收旧债
- 活跃 8-12：谨慎开坑，优先推进现有
- 活跃 < 8：可以正常开新坑

### 1.2.2 Hook年龄（陈旧度）
**计算公式**：当前章节 - 最近推进章节

| 年龄 | 状态 | 处理 |
|------|------|------|
| 0-3章 | 新鲜 | 正常推进 |
| 4-7章 | 正常 | 需要提及 |
| 8-10章 | 陈旧 | 必须推进或回收 |
| >10章 | 危险 | 必须立即处理 |

### 1.2.3 Hook推进频率
**健康标准**：每章至少推进1-2个活跃Hook

**检测**：
- 连续3章未推进某个Hook → 标记为停滞
- 连续5章未推进 → 强制要求推进或回收

### 1.2.4 Hook类型分布
**避免**：同一类型Hook过多

| 类型 | 建议最大数量 |
|------|-------------|
| 身份之谜 | 2-3个 |
| 力量来源 | 1-2个 |
| 恩怨情仇 | 2-3个 |
| 世界观秘密 | 1-2个 |
| 人物关系 | 2-3个 |

## 1.3 Hook议程制定

每章写作前，制定Hook Agenda：

### 1.3.1 Must Advance（必须推进）
- 选择2个最陈旧的Hook
- 选择1个与当前剧情最相关的Hook

### 1.3.2 Eligible Resolve（可回收）
- 已推进3章以上
- 近期有重要进展
- 读者期待度高

### 1.3.3 Stale Debt（陈旧债务）
- >10章未推进
- 必须在本章处理
- 可选择回收或延后

### 1.3.4 Avoid New（避免新开）
- 活跃Hook ≥ 10时，禁止开新坑
- 已有同类型Hook时，谨慎开坑

## 1.4 Hook回收时机判断

### 1.4.1 可以回收的信号
1. 已推进3章以上
2. 读者期待度达到峰值
3. 回收不会导致剧情断层
4. 回收后仍有其他Hook支撑

### 1.4.2 应该延后的信号
1. 回收会暴露过多信息
2. 更好的回收时机在后面
3. 当前有其他更重要的Hook需要处理
4. 回收后会导致Hook数量不足

## 1.5 Hook健康检查清单

每章结算时检查：

- [ ] 活跃Hook数量 ≤ 12
- [ ] 没有Hook年龄 > 10章
- [ ] 本章推进了至少1个Hook
- [ ] 没有同类型Hook过度堆积
- [ ] 陈旧债务得到处理

## 1.6 Hook常见问题

### 1.6.1 问题1：Hook遗忘
**症状**：伏笔埋了后不再提及
**解决**：
- 每章检查伏笔池
- 设置Hook Agenda
- 在章节摘要中记录Hook动态

### 1.6.2 问题2：Hook过度堆积
**症状**：活跃Hook > 15个
**解决**：
- 强制回收部分Hook
- 合并相关Hook
- 延后非关键Hook

### 1.6.3 问题3：Hook过早回收
**症状**：伏笔刚埋1-2章就回收
**解决**：
- 延长Hook生命周期
- 增加推进次数再回收
- 设置更长的回收时机

### 1.6.4 问题4：Hook类型单一
**症状**：全是"身份之谜"类Hook
**解决**：
- 多样化Hook类型
- 添加关系类、世界观类Hook
- 平衡各类Hook数量
